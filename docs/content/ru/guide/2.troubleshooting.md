## macOS говорит, что приложение «повреждено»

macOS помещает в карантин приложения, скачанные из интернета не через App Store. Чтобы исправить:

Откройте Терминал и выполните:

```bash
xattr -cr /Applications/LangSwitcher.app
```

Затем откройте приложение снова. Это нужно сделать только один раз.

Альтернативно: нажмите правой кнопкой мыши на приложении → **Открыть**.

## Не работает разрешение Accessibility

LangSwitcher требует разрешение Accessibility для чтения и замены текста.

1. Откройте **Системные настройки → Конфиденциальность и безопасность → Универсальный доступ**
2. Убедитесь, что LangSwitcher в списке и **включён**
3. Если уже был включён — попробуйте **выключить и снова включить**
4. Перезапустите LangSwitcher

::callout{icon="i-lucide-alert-triangle" color="amber"}
После обновлений macOS разрешения Accessibility иногда сбрасываются. Возможно, потребуется выдать разрешение заново.
::

## Неправильный результат преобразования

Такое может происходить с текстом, содержащим символы из нескольких раскладок. Попробуйте:

1. **Вручную выделите** конкретный текст, затем нажмите горячую клавишу
2. **Переключите режим умного преобразования** в Настройках → Основные (попробуйте «Последнее слово» или «Отключено»)
3. Проверьте, что нужные раскладки включены в **Системных настройках → Клавиатура → Источники ввода**

Если проблема сохраняется, [создайте issue на GitHub](https://github.com/reg2005/langSwitcher/issues) и приложите:
- Исходный текст
- Ожидаемый результат
- Фактический результат
- Ваши раскладки клавиатуры
- **Записи из журнала преобразований** (экспортируйте через Настройки → Журнал → Экспорт JSON)

## Горячая клавиша не работает в определённом приложении

Некоторые приложения (особенно работающие с повышенными привилегиями или в изолированных средах) могут не реагировать. Известные случаи:

- Terminal/iTerm при фокусе на командной строке
- Некоторые Electron-приложения с кастомными полями ввода

Попробуйте сначала выделить текст вручную, затем нажать горячую клавишу.

## Приложение не появляется в строке меню

1. Проверьте, что LangSwitcher запущен (посмотрите в Мониторе активности)
2. Иконка может скрываться за вырезом (notch) на MacBook Pro
3. Завершите и перезапустите приложение

## Высокая нагрузка на процессор

LangSwitcher должен потреблять минимум ресурсов. Если заметили высокую нагрузку:

1. Проверьте журнал — большие базы данных с записями могут замедлять работу
2. Попробуйте очистить журнал или установить лимит записей в Настройках → Журнал
3. [Сообщите о проблеме](https://github.com/reg2005/langSwitcher/issues) с данными из Монитора активности
